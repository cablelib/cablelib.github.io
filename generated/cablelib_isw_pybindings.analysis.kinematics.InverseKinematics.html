calculate(self: cablelib_isw_pybindings.analysis.kinematics.InverseKinematics, pose: cablelib_isw_pybindings.types.Pose) → numpy.ndarray[numpy.float64[m, 1]]

Does the inverse kinematics of the CDPR.

Given a pose this method calculates the resulting cable lengths. This can be done rather trivially as the vector representing a cable can be computed by adding vectors. The length of the cable is the Euclidian norm of the cable vector.

Refer to Pott, A. Cable-Driven Parallel Robots. Chapter 4.2.1 for more details.

Parameters:

pose (cablelib_isw_pybindings.types.Pose) – Pose for which the cable lengths are to be calculated

Returns:

Vector with all cable lengths.
