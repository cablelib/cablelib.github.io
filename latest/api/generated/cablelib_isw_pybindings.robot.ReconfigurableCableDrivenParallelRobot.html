ReconfigurableCableDrivenParallelRobot¶

class ReconfigurableCableDrivenParallelRobot¶

Bases: object

Wrapper that enables geometric reconfiguration of a cable-driven parallel robot.

The underlying library models a CDPR with an immutable BaseFrame. Reconfiguration is implemented by producing new immutable geometry snapshots. This class owns a home (reference) robot snapshot that never changes, a current robot snapshot that reflects the most recently applied reconfiguration, and a per-axis mapping that converts scalar axis positions into base anchor point coordinates.

A reconfiguration axis is interpreted as follows:

  • Each axis i has a home anchor point home_anchors_[i].

  • Each axis i has a direction vector axis_directions_[i].

  • Each axis i has a scalar position axis_positions_[i].

  • The resulting anchor point is computed as anchor_i = home_anchors_[i] + axis_directions_[i] * axis_positions_[i].

An axis is considered disabled if its direction vector is the zero vector, that is, if its norm equals zero. Disabled axes do not move their anchors, and their requested positions do not affect geometry.

If an update fails, for example due to limits violations, the wrapper keeps the previous state and raises an exception.

Constructs a reconfigurable wrapper around an existing home robot snapshot.

The home robot snapshot is used as the immutable reference configuration. The current snapshot is initialized to the home snapshot. All axis positions are initialized to zero.

Parameters:

  • home_robot – Immutable reference snapshot. Used as the baseline for all derived snapshots.

  • home_anchors – Home base anchor points, one per reconfiguration axis.

  • axis_directions – Direction vectors for each reconfiguration axis. A zero vector disables an axis.

  • limits – Scalar position limits for each axis.

Returns:

If metadata sizes do not match, or if the home robot’s base

frame attachment point count does not match the size of @p homeAnchors.

Return type:

ValueError

axis_positions(self) → list[float]¶

Returns the current scalar positions of all reconfiguration axes.

The returned vector has one entry per axis in the same order as home_anchors and axis_directions passed to the constructor.

current_robot(self) → CableDrivenParallelRobot¶

Returns the current robot snapshot.

The returned object reflects the most recently applied reconfiguration. It remains valid as long as this wrapper object remains alive.

reset_to_home(self) → None¶

Resets the current snapshot back to the home snapshot.

This restores all base anchors to their home positions and sets all stored axis positions to zero.

set_axis_position(self, axis_index: int, position: float) → None¶

Sets a single axis position and updates the current robot snapshot.

This is a convenience wrapper around @ref setAxisPositions. The update remains transactional.

Parameters:

  • axis_index – Index of the axis to update.

  • position – New position for the selected axis.

Errors:

ValueError: If axis_index is not a valid axis index. ReconfigurationError: If the requested position violates limits.

set_axis_positions(self, positions: Sequence[float]) → None¶

Sets all axis positions and updates the current robot snapshot accordingly.

This operation is transactional. The wrapper validates:

  • The vector length matches the number of axes.

  • Every enabled axis position lies within its configured limits.

If validation passes, the wrapper computes new base anchor points using the mapping described in the class documentation and produces a new current robot snapshot.

Parameters:

  • positions (positions New axis)

  • axis. (one per)

Errors:

ValueError: If positions has incorrect length. ReconfigurationError: If any enabled axis violates its limits.
